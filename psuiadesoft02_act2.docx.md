  
Actividad 2\. Generación de entornos de pruebas y casos de test asistidos por IA

**Objetivos**

Evaluar la capacidad del alumnado para definir, generar y ejecutar pruebas automatizadas sobre una aplicación desarrollada previamente, empleando únicamente asistentes de IA para la creación del código de test y la configuración del entorno de pruebas.

**Herramientas de referencia**  
Se podrán emplear asistentes de IA con capacidades de generación de código, tales como:

* ChatGPT con Code Interpreter o extensiones de desarrollo.  
* GitHub Copilot Chat.  
* Visual Studio Agent Mode.  
* IntelliJ Junie.  
* Cualquier otra herramienta vista en clase.

Frameworks y librerías de pruebas recomendados:

* Pytest, unittest (Python)  
* Junit (Java)  
* Jest, Mocha (JavaScript/Node.js)  
* Selenium, Playwright, Cypress (pruebas de interfaz web)

**Pautas de elaboración**

Partiendo de la aplicación desarrollada en la actividad anterior, el alumnado deberá crear un **entorno completo de pruebas** que cubra las funcionalidades clave de la aplicación.

**Restricciones:**

* Todo el código de pruebas debe ser generado mediante interacción con un asistente de IA.  
* El alumnado no escribirá código manualmente; cualquier implementación o corrección se solicitará al asistente.  
* El proceso debe documentarse con capturas y transcripciones de las interacciones más relevantes.

**Tareas** 

1. **Análisis de la aplicación existente**  
   * Identificar funcionalidades críticas que requieren validación.  
   * Definir el alcance y objetivos de las pruebas.  
2. **Interacción inicial con la IA**  
   * Redactar *prompts* para generar un entorno de pruebas adecuado.  
   * Solicitar scripts para instalar dependencias y configurar el framework de testing.  
3. **Generación de casos de prueba**  
   * Formular peticiones a la IA para obtener pruebas unitarias, de integración y, si procede, de interfaz.  
   * Ajustar y depurar los test mediante iteraciones con el asistente.  
4. **Ejecución y verificación**  
   * Ejecutar las pruebas en el entorno generado.  
   * Solicitar a la IA mejoras o refactorizaciones cuando haya fallos o resultados inesperados.  
5. **Documentación**  
   * Incluir un resumen del proceso.  
   * Presentar evidencias de ejecución (logs, capturas de pantalla).  
   * Reflejar las interacciones clave con el asistente.

**Extensión y formato**

* Informe final en PDF con:  
  * Resumen del alcance de las pruebas.  
  * Capturas y logs de ejecución.  
  * Conversaciones relevantes con la IA.  
  * Análisis de resultados y conclusiones.  
* Código de pruebas: en repositorio público o comprimido .zip.  
* Instrucciones de ejecución del entorno de pruebas (README.md).  
* Archivo requirements.txt o equivalente con dependencias.
