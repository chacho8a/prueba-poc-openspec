  
Actividad 1\. Desarrollo de aplicaciones con asistentes de programación basados en IA

**Objetivos**

Evaluar la capacidad del alumnado para diseñar, especificar y construir una aplicación de *software* interactuando de forma eficaz con un asistente de IA que genere e implemente el código necesario, sin realizar programación manual directa.

**Herramientas de referencia**  
Se podrán emplear asistentes de IA con capacidades de generación de código, tales como:

* ChatGPT con Code Interpreter o extensiones de desarrollo.  
* GitHub Copilot Chat.  
* Visual Studio Agent Mode.  
* IntelliJ Junie.  
* Cualquier otra herramienta vista en clase.

**Pautas de elaboración**

El alumnado deberá **diseñar y desarrollar una aplicación web funcional** que resuelva una necesidad concreta. Ejemplos de posibles aplicaciones:

* Gestor de tareas con persistencia en base de datos local.  
* Aplicación de reservas para un servicio ficticio.  
* Visualizador de datos en tiempo real desde una API pública.

**Restricciones:**

* Todo el código debe ser generado mediante interacción con un asistente de IA.  
* El alumnado debe guiar, especificar y depurar las respuestas del asistente hasta obtener un producto funcional.  
* No se permite escribir código manualmente; cualquier cambio debe ser solicitado al asistente.  
* Se puede emplear cualquier lenguaje de programación (Java, Python, etc.). 

**Tareas** 

1. **Definición del proyecto**  
   * Redactar un documento inicial con la descripción funcional de la aplicación.  
   * Establecer requisitos mínimos y funcionalidades esenciales.  
2. **Interacción con el asistente de IA**  
   * Formular *prompts* claros y específicos para generar código.  
   * Solicitar mejoras o correcciones según los resultados obtenidos.  
   * Documentar las conversaciones clave con el asistente.  
3. **Desarrollo**  
   * Integrar el código proporcionado por la IA.  
   * Iterar el proceso hasta lograr un prototipo estable.  
4. **Documentación del proceso**  
   * Incluir transcripciones relevantes de las interacciones con la IA.  
   * Justificar las decisiones tomadas y los cambios solicitados.  
   * Explicar las limitaciones encontradas.  
5. **Entrega final**  
   * Aplicación completamente funcional.  
   * Informe técnico que documente el proceso y la interacción con la IA.

**Extensión y formato**

* Informe final en PDF con:  
  * Descripción del proyecto.  
  * Requisitos iniciales.  
  * Capturas de pantalla de la aplicación funcionando.  
  * Extractos relevantes de las conversaciones con la IA.  
  * Análisis de resultados y conclusiones.  
* Código fuente: en un repositorio público (GitHub, GitLab, etc.) o comprimido en .zip.  
* Aplicación ejecutable o desplegada (enlace si se aloja *online* o instrucciones para ejecutar en local).  
* Archivo requirements.txt o equivalente con dependencias.
